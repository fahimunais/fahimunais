Project Overview

Built a production voice agent for banking, debt collection, and support flows using an open‑source stack. Focused on low latency, reliability, and safe responses while keeping costs and deployment footprint small.

Tech Stack

  • LLM: Ollama (Mistral)
  • STT: Whisper (streaming)
  • TTS: PiperTTS (phrase cache, denoise)
  • Telephony: Asterisk PBX (SIP/RTP, IVR)

Call Flow

Caller → Asterisk (SIP/RTP)
  → Whisper STT (streaming)
  → Policy / Intent gating
  → (Optional) Retrieval for facts
  → LLM (Ollama Mistral, tool-calls)
  → PiperTTS (cached phrases)
  → Asterisk playback → Next turn

Architecture

[Phone/SIP]
   |
[Asterisk PBX] -- RTP --> [Audio I/O]
                      |              
                 [Whisper STT]   [PiperTTS]
                      |              ^
                      v              |
               [Policy/Orchestrator] +--> [Tools/APIs]
                      |                    [RAG Store]
                      v
                 [Ollama (Mistral)]

Challenges & Mitigations

  • Latency: streaming STT; short prompts; tool-calls; phrase TTS cache; parallel prep.
  • Hallucinations: retrieval guardrails; refusal policy; intent gating for critical actions.
  • Voice quality (HW limits): Piper model choice; higher sample rate when available; light post‑FX.

Status & Outcomes

  • Deployed across multiple use cases (banking, collections, FAQs).
  • On‑prem friendly, OSS stack; tunable latency/quality trade‑offs.
